Deterministic Harness · 所有非 LLM 的可靠性层

什么是 Harness？

Harness 是 Ironsmith 架构中 所有非 LLM 的代码层。它们用确定性规则（正则、AST 操作、配置对象）确保系统的可靠性。Harness 不懂代码语义，但 100% 可靠。

🔧 核心原则： 凡是能用代码规则解决的，绝不用 LLM。LLM 只在 "需要理解代码语义" 的场景使用（如修复编译错误）。

所有 Harness 汇总

Harness ContentViewSourceCleanup

源码清洗。剥离 Markdown 围栏、移除 Preview 块、修复陷阱、格式化。

阶段 4 · 纯正则 + AST 操作 · 100% 确定

Harness ToolGenerationRepairPolicy

策略配置。定义所有数字阈值和预算（最大尝试次数、稳定阈值、无效 patch 阈值）。

阶段 5 · 配置对象 · 防止无限循环

Harness DeterministicRepairRules

确定性修复。基于正则规则修复常见 Swift/SwiftUI 编译错误。

阶段 5 · 正则规则库 · 无副作用

Harness SwiftPackageProcessClient

编译客户端。执行 swift build，解析编译器输出为结构化诊断。

阶段 5 · 进程调用 · 错误提取

Harness ToolVersionBackupClient

版本备份。原子暂存、promote、恢复、discard。

编辑模式 · 文件系统操作 · 路径逃逸检查

Harness ToolAppBundleClient

应用打包。签名、沙盒、图标、原子替换、quarantine 清除。

阶段 6 · 系统调用 · 安全发布

Harness ToolIconClient

图标生成。使用 Apple Foundation Model 或缓存生成应用图标。

阶段 6 · 图像生成 · 优雅降级

Harness AgentDiagnosticsLog

诊断日志。DEBUG 模式下记录修复循环的每一步（不存储完整模型响应）。

全局 · 日志记录 · 紧凑

Harness vs. Agent 对比

维度 Harness Agent
驱动力 代码规则（正则、配置、进程调用） LLM（生成文本）
可靠性 100% 确定（同样的输入总是同样的输出） 概率性（同样的输入可能不同输出）
成本 零 API 成本（本地计算） 高 API 成本（每次调用消耗 token）
速度 毫秒级 秒级（取决于模型响应时间）
能力 只能处理预定义模式 能处理未知模式（理解语义）
安全 安全（不会执行危险操作） 需要沙盒保护（可能生成危险代码）

对 LongMarch 的架构建议

基于 Ironsmith 的 Harness 设计，LongMarch 应该添加以下 Harness 层：

LongMarch 建议的 Harness 架构
// 1. HtmlSourceCleanup (源码清洗)
//    - 剥离 Markdown 围栏
//    - 移除 <html>/<head>/<body> 标签
//    - 移除外部 CSS/JS 引用
//    - 修复常见 HTML 错误

// 2. HtmlValidationPolicy (验证策略)
//    - 定义最大文件大小
//    - 定义禁止标签（script src, iframe, form action 等）
//    - 定义必需的 CSP 策略

// 3. DeterministicHtmlFixes (确定性修复)
//    - 自动闭合未闭合标签
//    - 修复错误嵌套
//    - 移除危险属性（onclick=eval, onerror 等）

// 4. AppBundleBuilder (应用打包)
//    - 将 HTML 嵌入 iframe 沙盒
//    - 添加 CSP 头
//    - 生成缩略图预览

// 5. VersionBackupClient (版本备份)
//    - 编辑前保存到数据库
//    - 支持一键恢复
//    - 保留最近 N 个版本（如 5 个）
