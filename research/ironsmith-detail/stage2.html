阶段② · 提示词优化

Prompt Refinement · 可选阶段 · 可跳过

首页 ① 元数据 ② 提示词优化 ③ 核心代码 ④ 源码清洗 ⑤ 编译修复 ⑥ 应用打包 编辑模式
Agent（LLM 驱动）· 可选

职责

将用户的模糊描述扩展为详细的实现指令。这是一个 可选阶段，由 promptRefinementEnabled 标志控制。如果用户描述很清晰，可以直接跳过。

源码结构

// PromptRefinement.swift
// 职责：将模糊用户描述扩展为详细实现指令

struct PromptRefinement {
    let isEnabled: Bool  // promptRefinementEnabled
    let languageModel: LanguageModel
    
    func refine(_ userPrompt: String) async -> String {
        guard isEnabled else {
            return userPrompt  // 直接跳过，返回原始描述
        }
        
        let systemPrompt = """
        You are a product requirements translator. Given a user's vague
        app description, expand it into detailed implementation instructions
        while preserving the user's intent.
        """
        
        let refinementPrompt = """
        User wants to build an app. Expand their description into detailed
        implementation instructions. Preserve their intent exactly.
        
        User description: \(userPrompt)
        
        Provide a refined description that includes:
        - Core functionality and UI layout
        - Key user interactions
        - Data flow (if any)
        - Visual style preferences (if implied)
        
        Do NOT add features the user didn't mention.
        Do NOT change the scope or complexity.
        """
        
        let refined = await languageModel.chat(
            system: systemPrompt,
            user: refinementPrompt
        )
        
        return refined
    }
}

模型提示词

系统提示（System Prompt）
You are a product requirements translator for a macOS utility app builder.
Your job is to take a user's vague or brief description and expand it into
clear implementation instructions for a SwiftUI developer.

Rules:
- Preserve the user's exact intent — do not add features they didn't mention
- Do not increase scope or complexity beyond what they asked
- Focus on what the app DOES, not how it's built
- Include UI layout suggestions only if the user implied them
- Keep it concise but complete (3-5 sentences max)
用户提示（User Prompt）示例
User wants to build an app. Expand their description into detailed
implementation instructions. Preserve their intent exactly.

User description: "A clipboard cleaner that strips tracking parameters"

Provide a refined description that includes:
- Core functionality and UI layout
- Key user interactions
- Data flow (if any)
- Visual style preferences (if implied)
预期输出（Expected Output）
A menubar utility that monitors the system clipboard. When a URL is
copied, it automatically strips common tracking parameters (utm_source,
utm_medium, fbclid, etc.) and replaces the clipboard with the clean URL.
The UI should show a small window listing recent cleaned URLs with
original and cleaned versions. Include a toggle to enable/disable
auto-cleaning. Minimalist dark theme design.

关键设计决策

为什么是可选的？

避免过度工程化。 如果用户描述已经很清晰（如 "Build a calculator with basic operations"），再让 LLM "优化" 可能会添加不必要的功能（如 "add scientific mode"）。所以默认关闭，只在用户描述模糊时开启。

为什么强调 "不要添加未提及的功能"？

LLM 有强烈的 "helpful" 倾向，会主动添加它认为 "有用" 的功能。这在代码生成中可能导致过度复杂。明确的负约束防止 LLM "好心办坏事"。

💡 对 LongMarch 的启示： 可以考虑在 Generator 页面添加一个 "Auto-enhance description" 开关，让用户选择是否启用提示词优化。默认关闭，避免意外添加功能。
